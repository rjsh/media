41/*****************************************************************************
42******************************************************************************
43
44 Theory of operation
45
46
47 The Device Interface Library (DIL) allows application level code, such
48as a DirectShow filter, to access the Broadcom CrystalHD decoder driver to
49provide hardware decoding for MPEG-2, H.264 (AVC) and VC-1 streams.
50
51 In the Microsoft DirectShow system, the overall system graph would look
52like the following:
53
54+--------+ +---------------+ +---------------+ +--------------------+
55| Source |->| Demultiplexer |->| Audio decoder |->| DirectSound Device |
56+--------+ +---------------+ +---------------+ +--------------------+
57 |
58 | +-------------------------+ +----------------+
59 +->| Broadcom decoder filter |->| Video Renderer |
60 +-------------------------+ +----------------+
61 | |
62 +----------------+
63 | Broadcom DIL |
64 +----------------+
65 | |
66 +-----------------+
67 | Broadcom Driver |
68 +-----------------+
69
70 From the view of the caller, the DIL will accept compressed video streams
71and will output decoded video frames or fields to seperate Y and UV buffers.
72The DIL is responsible solely for decoding video and has no responsibilities
73for audio nor for rendering, as shown in the above diagram. Audio/video
74sychronization is assisted by feeding the DIL with timestamps so that it
75may pass those timestamps along with the decoded video. The timestamped
76output video will then be presented at the appropriate time by the renderer.
77
78A minimal implementation would be:
79
80 HANDLE hBRCMhandle;
81 uint8_t input_buffer[INPUT_SIZE];
82 uint8_t y_output_buffer[WIDTH*HEIGHT];
83 uint8_t uv_output_buffer[WIDTH*HEIGHT];
84 BC_DTS_PROC_OUT sProcOutData = { fill in your values here };
85 BC_PIC_INFO_BLOCK sPIB = { fill in your values here };
86
87 // Acquire handle for device.
88 DtsDeviceOpen(&hBRCMhandle, 0);
89
90 // Elemental stream.
91 DtsOpenDecoder(hBRCMhandle, 0);
92
93 // H.264, Enable FGT SEI, do not parse metadata, no forced progressive out
94 DtsSetVideoParams(hBRCMhandle,0,1,0,0,0);
95
96 // Tell decoder to wait for input from host. (PC)
97 DtsStartDecoder(hBRCMhandle);
98
99 // Input buffer address, input buffer size, no timestamp, Unencrypted
100 DtsProcInput(hBRCMhandle,input_buffer,sizeof(input_buffer),0,0);
101
102 // Tell PC to wait for data from decoder.
103 DtsStartCapture(hBRCMhandle);
104
105 // 16ms timeout, pass pointer to PIB then get the decoded picture.
106 DtsProcOutput(hBRCMhandle,16,&sPIB);
107
108 // Stop the decoder.
109 DtsStopDecoder(hBRCMhandle);
110
111 // Close the decoder
112 DtsCloseDecoder(hBRCMhandle);
113
114 // Release handle for device.
115 DtsDeviceClose(hBRCMhandle);
116

407/*****************************************************************************
408
409Function name:
410
411 DtsSetVideoParams
412
413Description:
414
415 Sets various codec parameters that would be used by a subsequent call
416 to DtsStartDecoder.
417
418 DtsSetVideoParams must always be called before DtsStartDecoder for the
419 decoder to start processing input data. The device must have been
420 previously opened for this call to succeed.
421
422Parameters:
423 hDevice Handle to device. This is obtained via a prior call to
424 DtsDeviceOpen.
425 videoAlg Video Codec to be enabled to decode stream.
426 H.264 (0), VC-1 (4) and MPEG-2 (1) currently supported.
427 All other values will return BC_STS_INV_ARG
428 FGTEnable Enable processing of FGT SEI.
429 MetaDataEnable Enable retrieval of picture metadata to be sent to video
430 pipeline.
431 Progressive Instruct decoder to always try to send back progressive
432 frames. If input content is 1080p, the decoder will
433 ignore pull-down flags and always give 1080p output.
434 If 1080i content is processed, the decoder will return
435 1080i data. When this flag is not set, the decoder will
436 use pull-down information in the input stream to decide
437 the decoded data format.
438 OptFlags In this field bits 0:3 are used pass default frame rate,
439 bits 4:5 are for operation mode (used to indicate Blu-ray
440 mode to the decoder) and bit 6 is for the flag mpcOutPutMaxFRate
441 which when set tells the FW to output at the max rate for the
442 resolution and ignore the frame rate determined from the
443 stream. Bit 7 is set to indicate that this is single threaded mode
444 and the driver will be peeked to get timestamps ahead of time.
445
446Return:
447
448 BC_STS_SUCCESS will be returned on successful completion.
449
450*****************************************************************************/

915/*****************************************************************************
916
917Function name:
918
919 DtsProcInput
920
921Description:
922
923 Sends compressed (coded) data to the decoder for processing.
924
925 The device must have been previously opened for this call to succeed.
926 In addition, suitable keys must have been exchanged for decryption and
927 decode to be successful.
928
929Parameters:
930
931 hDevice Handle to device. This is obtained via a prior call to
932 DtsDeviceOpen.
933 pUserData Pointer to data buffer that holds the data to be transferred.
934 [INPUT]
935 sizeInBytes Size in Bytes of data available to be sent to the decoder for
936 processing.
937 Timestamp Optional timestamp information attached to the media sample
938 that is available in the buffer. If timestamp is present
939 (i.e. non-zero), then this will be reflected in the output
940 sample (picture) produced from the contents of this buffer.
941 Timestamp should be in units of 100 ns.
942 Encrypted Flag to indicate that the data transfer is not in the clear
943 and that the decoder needs to decrypt before it can decode
944 the data. Note that due to complexity, it is preferred that
945 the application writer uses the higher level
